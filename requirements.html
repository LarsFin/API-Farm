API Farm Requirements

Each endpoint for the API is listed below. Including its expected behaviour with given data.

GET /video_games

Retrieves all video games from storage and responded as an array in the body.

Examples

Request

Request of method type GET made to /video_games endpoint.

Response 200

[
    {
        "name": "Perfect Dark",
        ...
    },

    {
        "name": "Super Mario 64",
        ...
    }
]

GET /video_games/:id

Retrieves video game with specified id from storage and set as response body.

Examples

Request

Request of method type GET made to /video_games/:id, where the id is that of the desired video game.

Response 200

{
    "name": "Bomberman 64",
    ...
}

Response 404

No video game with id 'x' could be found.

POST /video_games

Creates a video game within storage with the provided data from the request body. Failing to provide required or invalid data will result in a bad request response. The table below shows what data is expected and its type.

Attribute Type Required? Default Value Description
name string yes N/A The name of the video game. This must be provided in the body of the request.
developers string[] no [] An array of the names of the developers who worked on the video game.
directors string[] no [] An array of the names of the people who directed the video game.
producers string[] no [] An array of the names of the people who produced the video game.
designers string[] no [] An array of the names of the people who designed the video game.
programmers string[] no [] An array of the names of the people who programmed the video game.
artists string[] no [] An array of the names of the people who worked on the art of the video game.
composers string[] no [] An array of the names of the people who composed the soundtrack of the video game.
platforms string[] no [] An array of the names of the platforms the video game had been released on.
date_released string yes N/A The date that the video game had been released. The date must be in the format 'DD/MM/YYYY'.

Response 400

A bad request can be the result of either not providing a value for the name or date_released. Additionally, a bad request can be responded when the provided date_released is invalid. 

A name is required for a video game.
The provided date_released '29 June 1998' is invalid and must be in the format 'DD/MM/YYYY'.
